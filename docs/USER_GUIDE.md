# 用户指南（当前可用功能）

> 适用版本：当前仓库实现（认证与首登重设流程已可用）

## 1. 使用前准备

1. 启动后端：`npm run dev:backend`
2. 启动前端：`npm run dev:frontend`
3. 打开浏览器：`http://localhost:3000`

---

## 2. 注册账号

前端页面路径：`/register`  
后端接口：`POST /api/auth/register`

操作步骤：
1. 访问 `/register`
2. 填写：用户名、邮箱、密码、学号
3. 点击“注册”
4. 成功后系统会自动登录并跳转到首页 `/`

输入要求（后端最终校验）：
- 用户名：3-50 个字符
- 邮箱：合法邮箱格式
- 密码：6-128 个字符
- 学号：11 位数字

---

## 3. 登录账号

前端页面路径：`/login`  
后端接口：`POST /api/auth/login`

操作步骤：
1. 访问 `/login`
2. 输入邮箱和密码
3. 点击“登录”
4. 成功后跳转到首页 `/`

登录后行为：
- 前端会保存登录 token（本地存储）
- 刷新页面时会调用 `GET /api/auth/me` 恢复登录状态

---

## 4. 管理员首次登录强制重设密码

当前后端启动时会自动确保存在默认系统管理员（若不存在则创建），该账号首次登录后必须先重设密码。

相关行为：
- 登录返回用户字段 `forcePasswordReset=true` 时
- 前端访问首页 `/` 会被自动跳转到 `/reset-password`
- 未完成重设前无法继续使用首页内容

前端页面路径：`/reset-password`  
后端接口：`POST /api/auth/reset-password`（需 Bearer Token）

操作步骤：
1. 使用管理员账号登录
2. 按提示进入 `/reset-password`
3. 输入新密码与确认密码并提交
4. 成功后 `forcePasswordReset` 变为 `false`，可正常访问首页

管理员新密码额外规则（system_admin）：
- 必须同时包含：大写字母、小写字母、特殊符号

---

## 5. 常见错误与处理

### 5.1 `Validation failed`
含义：请求参数未通过后端校验。  
处理：
- 检查字段是否缺失
- 检查格式（尤其邮箱、学号、密码长度）
- 重设密码时确认两次密码一致

常见触发场景：
- 学号不是 11 位数字
- 密码长度不足 6 位
- 管理员新密码不满足复杂度规则

### 5.2 `Email already registered`
含义：邮箱已被占用。  
处理：改用其他邮箱，或直接去 `/login` 登录。

### 5.3 `Username already taken`
含义：用户名已被占用。  
处理：更换用户名后重试注册。

### 5.4 `Invalid email or password`
含义：登录凭据错误。  
处理：确认邮箱与密码输入，必要时重新注册测试账号。

### 5.5 `Missing or invalid Authorization header` / `Invalid or expired token`
含义：未携带有效登录令牌或令牌已失效。  
处理：重新登录后再访问受保护页面或接口。

## 6. 当前功能范围（已实现）

- 用户注册
- 用户登录
- 登录态恢复（`/api/auth/me`）
- 受保护路由拦截（未登录跳转 `/login`）
- 系统管理员首次登录强制重设密码
- 基础首页（显示当前用户信息、可退出登录）
